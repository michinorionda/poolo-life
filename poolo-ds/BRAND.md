# POOLO ブランド & ライティング指針

`poolo-ds` デザインシステムの「思想」の部分。`tokens.css` / `components.css` が**見た目の単一ソース**なら、この BRAND.md は**言葉と間（ま）の単一ソース**。新しい告知ページ・サムネ・OG画像を作るとき、色やコンポーネントと同じ強さでここに従う。

正規リファレンス: `../brand-explore-260708/direction-b2-final.html` ／ by TABIPPO・POOLO。

---

## 1. ブランド基盤（Foundations）

| 項目 | 値 | 実装トークン |
|---|---|---|
| メインカラー（公式・主役） | `#189BBC` | `--brand` |
| 濃色（ホバー・強調・引用） | `#127E9A` | `--brand-deep` |
| 中間トーン（補助） | `#8FC6D6` | `--brand-mid` |
| 最淡の面（問い・写真足元） | `#DDECF1` | `--brand-pale` |
| 文字（純黒 / ソフト黒） | `#000000` / `#1E1E1E` | `--ink` / `--ink-soft` |
| 補助テキスト（黒アルファ） | `rgba(0,0,0,.72)`〜`.40` | `--ink-72 … --ink-40` |
| 背景（白 / オフホワイト） | `#FFFFFF` / `#F9F9F9` | `--bg` / `--bg-off` |
| 罫線（ヘアライン / 淡罫） | `#E4E4E4` / `#C2DCE5` | `--line` / `--line-pale` |
| 見出しの"顔"（明朝） | Zen Old Mincho | `--mincho` |
| データ／UI見出し | Zen Kaku Gothic New | `--head` |
| 本文 | Noto Sans JP | `--body` |
| 欧文（英字ラベル・数字） | Jost | `--en` |

**色数は絞る。** 主役は公式ブランドブルー `#189BBC` の**1色**。文字は黒、背景は白、区切りはヘアライン。ブルーの濃淡（deep/mid/pale）以外に色を足さない。旧デザインのくすみ深緑・水色・くすみ緑は使わない。

## 2. 声のトーン（Voice）

POOLO の言葉は **「煽らない・断定しない・急かさない・内省的」**。読み手を追い込まず、内側に問いを置いて帰ってもらう。

### やること
- **短い言い切りを句点で細かく区切り、リズムを作る。** 体言止めと「。」を混在させる。
  > 仕事は頑張ってきた。評価もそれなりにされていて、できることも増えてきている。
- **断定せず、問いに開く。** 口語の自問をそのまま置く。強調は `.quoted`（`--brand-deep`）で。
  > 「これで合っているんだっけ」／「〜って何だっけ…？」
- **転換は一言で、間を作る。** 「ただ、しかし。」のような一言を `.intro-break`（明朝・罫つき・大きめ）で独立させる。
- **CTA も誘い。** 命令形にしない。「気になった方は、お早めに。」のトーン。
- **内省系の語彙。** 「納得感」「内側の手触り」「言語化」「まだ出会ったことのない自分」。

### やらないこと
- 煽り・恐怖訴求・過度な数字訴求（「今すぐ！」「残りわずか！！」）。
- 事実（先着◯名・参加無料）は**静かに**提示する。強調しすぎない。
- 装飾過多。感嘆符の連打。

### 参加者の声の掲載ルール（`.voices`）
- **逐語で載せる。** 本人が書いたことばをそのまま。要約・脚色をしない（引用は明朝 `--mincho`）。
- **属性のみ添える。** 「メーカー・企画職／30代・体験会参加者」のように、業種・年代・立場だけ（属性は本文書体 `--body` で引用と質感差をつける）。
- **特定を避ける。** 氏名・所属企業名・顔写真は載せない。末尾に `.voice-note` で「ご本人が特定されない形で掲載しています」と明記する。

## 3. 視覚言語（Visual Language）

言葉と同じくらい「余白」と「面の設計」が POOLO らしさを作る。

- **ブランドブルーが主役。** ヘッダーCTA・日付帯・中間CTA・タグは `--brand` の**フラットなベタ面**。グラデーション・blur影は使わない（脱AI）。
- **構造はヘアライン基調。** 区切りは 1px の `--line`。影の代わりに罫線で面を立てる。角丸は 2px（`--radius`）だけ、乱発しない。
- **大見出しの顔は明朝。** Hero タイトル・`.sec-title`・引用・問いは Zen Old Mincho（`--mincho`）。データや小見出しは Zen Kaku（`--head`）、英字・数字は Jost（`--en`）で受ける。
- **写真ドリブン。** 写真は裁ち落とし（フルブリード）で、`--brand` の罫で額装（Hero は左8px、photo-break は足元にブルー淡面）。編集的なキャプション（英字ラベル＋罫＋和文）を添える。
- **非対称マガジン。** セクションは左に細いスパイン（連番 or 和文ラベル）、右に見出し＋本文の非対称グリッド。
- **級数コントラスト。** 日付の数字（clamp 44–76px）や見出しは大きく、ラベルは 10–11px＋広い字間で小さく。強弱を効かせる。
- **組版の作法。** 見出し・詰めたい箇所は `font-feature-settings: "palt"`。`.nb`（`white-space: nowrap`）で泣き別れ（行末禁則）を防ぐ。字間は本文 0.05em、ラベルは 0.2〜0.42em。

## 4. ページ構成の型（告知ページ）

正規リファレンス `direction-b2-final` が確立した標準の並び。新規ページはこの型を出発点に、不要セクションを引き算する：

1. **固定ヘッダー**（`.brand-row`、ロゴ＋申込CTA）
2. **写真ドリブンHero**（`.hero` / `.hero-copy` ＋ `.hero-photo`）
3. **ブルー日付帯**（`.hero-band`）
4. **イベント要点**（`.summary`、日時／形式／定員／参加費）
5. **お知らせバンド**（`.notice-bar`）
6. **01 導入**（`.sec` + `.intro-body` + `.intro-break`）
7. **写真ブレイク**（`.photo-break--right`）
8. **03 当日の流れ**（`.sec` + `.flow-list`、4フェーズ）
9. **当日の問い**（`.sec-questions` + `.q-list`）
10. **写真ブレイク**（`.photo-break--left`）
11. **参加者の声**（`.sec` + `.voice-list`）
12. **About POOLO**（`.about`）
13. **FAQ**（`.sec` + `.faq-list`）
14. **中間・最終CTA**（`.cta`）
15. **追従CTA**（`.float-apply`、要JS）
16. **フッター**（`.footer`、家紋＋2社ロゴ）

各コンポーネントの最小マークアップは `poolo-ds/README.md` とカタログ `poolo-ds/index.html`（`@dsCard`）を参照。

## 5. 運用の約束

- **トークンとコンポーネントはコピペしない。** 新規ページは `poolo-ds/tokens.css` と `poolo-ds/components.css` を `<link>` で読む（フォントの `@import` は tokens 側に1本化済み）。
- **新しい汎用パーツを作ったら DS に還元する。** ページに書きっぱなしにせず `components.css` / `tokens.css` / カタログ（`@dsCard`）/ README に反映する。
- **このトーンから外れる案は「素材」扱い。** ゼロイチの実験は歓迎だが、標準の告知ページはこの BRAND.md に収束させる。
