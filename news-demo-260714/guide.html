Member Guide

はじめての
サイトづくり・更新ガイド

Claude Code（クロードコード）を使って、自分のWebサイトをつくってお知らせを更新するまで。
プログラミングの知識はいりません。

💡 いちばん大事なこと

Claude Code は「AIにお願いすると、代わりに作業してくれる道具」です。コードは書きません。ふつうの日本語で「〇〇して」とお願いするだけ。この1点さえ分かっていれば大丈夫です。

この順番で進めます
  1. 最初の準備（一度だけ）
  2. 基本の使い方をおぼえる
  3. サイトをつくる
  4. お知らせを更新する（いちばん使う）
  5. 変更を公開する（リンクで共有）
  6. 困ったときは
1

最初の準備（一度だけ）

ここが一番むずかしいところ。最初はサポート担当と画面共有しながら一緒にやるのがおすすめです。一度できれば、次からは不要です。

🧰 用意するもの（2つ）

  • Claude の有料プラン（Pro または Max） … Claude Code を使うために必要です。
  • GitHub（ギットハブ）のアカウント … サイトのファイルを保管＆公開する場所。無料で作れます。

🚪 Claude Code の始め方（一番かんたんな方法）

  1. ブラウザで claude.ai/code を開く
  2. Claude のアカウント（Proと同じメール）でサインイン
  3. 画面の案内にそって GitHub と接続（「Claude の GitHub App」を許可）
  4. 作業する場所（リポジトリ）を選べば準備完了
💡 ヒント：パソコンにアプリを入れたい場合は「Claude デスクトップアプリ」でもOK。ターミナル（黒い画面）は使わなくて大丈夫です。まずは ブラウザ版（claude.ai/code）が一番かんたんです。
⚠️ この準備だけは要サポート：GitHub アカウントや公開設定（GitHub Pages）まわりは、最初の一度だけ、詳しい人と一緒に進めると安心です。
2

基本の使い方をおぼえる

やることは「お願いする → 提案を確認する → OKする」の3つだけ。

💬 ① 日本語でお願いする

チャット欄に、やってほしいことをそのまま書きます。

トップページに「お問い合わせ」ボタンを追加して。

👀 ② 提案（変更内容）を確認する

Claude Code は、変更する前に「どこをどう変えるか」を見せてくれます。緑＝追加、赤＝削除。内容を見て…

✅ ③ OK（承認）する

  • よければ Accept（承認）、やり直してほしければ Reject（却下） を押します。
  • 作業を途中で止めたいときは Esc キー。
  • 「さっきの状態に戻したい」ときは /rewind と入力して、戻したい時点を選びます。
💡 安心ポイント：勝手にどんどん進むことはありません。大事な操作は必ず「確認 → あなたがOK」の順。まずは気軽に試して大丈夫です。
3

サイトをつくる

「こんなサイトが欲しい」を伝えれば、たたき台を作ってくれます。

たとえば、このデモサイトを見せながらこう頼めます：

このデモ（https://michinorionda.github.io/poolo-life/news-demo-260714/）みたいな、やさしい雰囲気のフリースクールのサイトを作って。トップ・お知らせ一覧・お問い合わせのページが欲しいです。
  • 色・雰囲気の希望があれば一緒に伝える（例：「青系でやさしい感じ」）
  • 載せたい文章や写真があれば「これを使って」と渡す
  • できあがりを見て「ここをこう直して」と何度でもお願いできます
💡 写真も渡せます：使いたい画像をパソコンのデスクトップに置いて、「デスクトップの〇〇.png をトップの写真に使って」と伝えればOK。Claude Code は画像を見て判断できます。
4

お知らせを更新する（いちばん使う操作）

ふだんの運用は、ほぼこれ。やり方は2通りあります。

🟢 方法A：Claude Code にお願いする（おすすめ）

いちばんかんたん。日本語で伝えるだけ。

8月1日に「夏の親子ワークショップを開催します」というお知らせ（イベント）を追加して。本文は、8月上旬に工作やゲームをして過ごす会で、対象は小中学生、事前予約制、という内容でお願い。

画像も足したいときは：

その記事のメイン画像に、デスクトップの natsu-event.png を使って。

🔵 方法B：自分でファイルに書き足す

仕組みを知っておくと安心。お知らせは news-data.js（見出し）と articles.js（本文）の2つに、同じ「id（目印）」で書き足すだけです。デザインのファイルは触りません。

💡 迷ったら方法Aでお願いすれば、Claude Code がこの2ファイルを正しく書き換えてくれます。「自分で触るのはこわい」人は方法Aだけで十分です。

✏️ よく使うお願い（コピペで使えます）

〇月〇日の「△△」というお知らせを追加して。カテゴリは（お知らせ / イベント / メディア）。
〇〇の記事の日付を△月△日に直して。
一番古いお知らせを1件消して。
トップの写真を、デスクトップの □□.png に変えて。
5

変更を公開する（リンクで共有）

編集しただけでは、まだ自分の画面の中だけ。「公開して」と伝えると、みんなが見られるサイトに反映されます。

いまの変更をサイトに反映して（公開して）。
  • Claude Code が「保存して公開」（専門用語で commit / push）まで代わりにやってくれます。
  • 反映には数分かかることがあります。少し待ってからページを再読み込み（更新）しましょう。
  • 公開されたサイトのURLを、そのまま案内や告知に貼れます。
⚠️ 公開すると誰でも見られる状態になります。個人情報や、まだ見せたくない内容が含まれていないか、公開前に一度確認しましょう。
6

困ったときは

つまずいても大丈夫。だいたいこれで解決します。

🗣️ うまく伝わらないとき

  • 具体的に：どのページの・どこを・どうしたいか。（例：「トップの、お知らせの下に、地図を追加して」）
  • うまくいかない画面はスクリーンショットを貼ると伝わりやすい。
  • エラーの文が出たら、そのまま貼りつけて「これはどういう意味？直して」と聞けばOK。

↩️ もとに戻したいとき

  • 作業を止める：Esc
  • 前の状態に戻す：/rewind → 戻したい時点を選ぶ

🧹 別の作業を始めるとき

  • 話題を大きく変えるときは /clear と入力すると、頭がスッキリして精度が上がります。
📖 用語ミニ辞典（わからない言葉が出たら）
リポジトリ
サイトのファイル一式を保管する「箱」。GitHub の中にあります。
コミット / commit
変更に「ここまでやった」と印をつけて保存すること。
プッシュ / push
保存した変更を、公開用の場所に送ること。
GitHub Pages
そのファイルを「Webサイト」として無料で公開してくれる仕組み。
Accept / Reject
Claude の提案を「OK」か「やり直し」か選ぶボタン。
